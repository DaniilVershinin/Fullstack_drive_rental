# DriveGO

Fullstack-приложение для аренды автомобилей: каталог, бронирование, личный кабинет клиента, панель менеджера и админ-панель. Данные берутся из Supabase, а если переменные окружения не заданы, приложение остается работоспособным на локальных fallback-данных.

## Стек

- React 18, TypeScript, Vite
- React Router
- Tailwind CSS + локальные CSS-компоненты
- Supabase: Auth, PostgreSQL, RLS
- Vitest + Testing Library

## Запуск локально

```bash
npm install
npm run dev
```

Откройте `http://localhost:5173`.

Для проверки:

```bash
npm run build
npm run test:run
```

## Деплой на Vercel

Проект готов к деплою как Vite-приложение. В репозитории есть `vercel.json`, который перенаправляет все маршруты на `index.html`, чтобы страницы React Router вроде `/catalog`, `/cabinet`, `/manager` и `/admin` открывались напрямую после обновления страницы.

Настройки проекта на Vercel:

- Framework Preset: `Vite`
- Build Command: `npm run build`
- Output Directory: `dist`
- Install Command: `npm install`

В `Project Settings` → `Environment Variables` добавьте:

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key
```

После изменения переменных окружения сделайте новый deploy, чтобы Vercel пересобрал фронтенд с актуальными значениями.

## Переменные окружения

Создайте файл `.env` в корне проекта:

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key
```

Где взять значения:

1. Откройте проект в Supabase.
2. Перейдите в `Project Settings` → `API Keys`.
3. В `.env` вставьте `Project URL`.
4. В `.env` вставьте ключ из блока `Publishable key`.

Секретный ключ `service_role` во фронтенд добавлять нельзя.

## Настройка Supabase

1. Создайте проект в Supabase.
2. Откройте `SQL Editor`.
3. Запустите [supabase/schema.sql](./supabase/schema.sql).
4. Запустите [supabase/seed_more.sql](./supabase/seed_more.sql), если нужен расширенный набор машин, городов и характеристик.
5. Создайте пользователей через `Authentication` → `Users` → `Add user`.
6. После создания пользователей откройте [supabase/seed_users_orders_template.sql](./supabase/seed_users_orders_template.sql), подставьте реальные UUID пользователей и запустите SQL для тестовых заказов.

Если SQL пишет `Success. No rows returned`, это нормально: скрипт создал/обновил таблицы, политики или данные, но не обязан возвращать строки.

## Роли

Роль хранится в таблице `profiles` в колонке `role`:

- `client` — личный кабинет и бронирование.
- `manager` — панель менеджера `/manager`, управление статусами заказов и доступностью машин.
- `admin` — админ-панель `/admin`, управление машинами, пунктами выдачи, пользователями и тарифами.

Чтобы выдать роль вручную:

```sql
update public.profiles
set role = 'admin'
where email = 'you@example.com';
```

Для менеджера:

```sql
update public.profiles
set role = 'manager'
where email = 'manager@example.com';
```

## Функционал по ролям

### Клиент

Клиентский сценарий покрывает полный путь аренды автомобиля: от выбора параметров поездки до оформления бронирования и работы с личным кабинетом.

Доступный функционал:

- Регистрация нового аккаунта через форму авторизации.
- Валидация регистрации: обязательное имя, корректный email, пароль от 8 символов, дата рождения, минимальный возраст 23 года, номер водительского удостоверения.
- Вход в аккаунт через Supabase Auth или демо-аккаунт при незаданном Supabase.
- Просмотр главной страницы с быстрым подбором автомобиля по городу, классу и датам аренды.
- Выбор города из списка доступных городов.
- Выбор класса автомобиля: любой, эконом, комфорт, бизнес, SUV.
- Выбор периода аренды через даты или быстрые пресеты на несколько дней.
- Переход из главной формы сразу в каталог с выбранными параметрами.
- Просмотр каталога автомобилей.
- Фильтрация каталога по городу.
- Фильтрация по категории автомобиля.
- Фильтрация по типу топлива.
- Фильтрация по коробке передач.
- Фильтрация по приводу.
- Фильтрация по минимальной мощности.
- Фильтрация по максимальной цене за сутки.
- Фильтрация только свободных автомобилей.
- Сброс всех фильтров каталога.
- Сортировка автомобилей по возрастанию цены.
- Сортировка автомобилей по убыванию цены.
- Сортировка по новизне автомобиля.
- Сортировка по мощности.
- Сортировка по названию.
- Просмотр карточек автомобилей с фото, статусом доступности, городом, годом, топливом, категорией, мощностью, объемом двигателя, приводом и ценой за сутки.
- Переход на детальную страницу автомобиля.
- Просмотр подробных характеристик автомобиля: категория, год, топливо, коробка передач, привод, мощность, объем двигателя, кузов, цвет, объем багажника, количество мест, город и штраф за день просрочки.
- Выбор дат аренды на странице автомобиля.
- Автоматическая проверка занятости автомобиля на выбранные даты.
- Запрет бронирования автомобиля, если он уже занят на выбранный период.
- Запрет бронирования автомобиля со статусом `busy`.
- Просмотр предварительной стоимости аренды без дополнительных услуг.
- Открытие модального окна бронирования из каталога или со страницы автомобиля.
- Оформление бронирования в 3 шага: параметры, дополнительные услуги, оплата.
- Выбор даты начала и даты окончания аренды.
- Быстрый выбор длительности аренды через пресеты.
- Выбор пункта выдачи автомобиля.
- Выбор пункта возврата автомобиля, включая вариант "тот же пункт".
- Выбор дополнительных услуг из списка Supabase или fallback-данных.
- Автоматический пересчет итоговой стоимости с учетом срока аренды и дополнительных услуг.
- Выбор способа оплаты: банковская карта, СБП или оплата при получении.
- Валидация карточной оплаты: номер карты, срок действия, CVV, имя на карте.
- Создание заказа в Supabase или fallback-режиме.
- Получение уведомления об успешном бронировании.
- Просмотр номера подтверждения бронирования.
- Просмотр подсказки о документах, которые нужно взять с собой.
- Просмотр личного кабинета.
- Просмотр статистики в личном кабинете: текущие заказы, завершенные заказы, отмененные заказы, общая оплаченная сумма.

- Повтор бронирования: переход обратно в каталог с параметрами города и категории выбранного автомобиля.
- Редактирование профиля клиента.
- Обновление имени, фамилии, телефона, даты рождения и номера водительского удостоверения.

## Где задавать картинки машин

Вариант через интерфейс:

1. Войдите под пользователем с ролью `admin`.
2. Откройте `/admin`.
3. Перейдите во вкладку `Автомобили`.
4. Выберите машину из списка или создайте новую.
5. В поле `Фото URL` вставьте прямую ссылку на изображение.
6. Нажмите `Сохранить авто`.

Вариант через SQL:

```sql
update public.cars
set photo_url = 'https://example.com/car.jpg'
where name = 'Toyota Camry';
```

Лучше хранить изображения в Supabase Storage или другом CDN. Если `photo_url` пустой или картинка не загрузилась, приложение покажет локальный SVG fallback.

## Supabase Storage для фото

1. Откройте `Storage`.
2. Создайте bucket, например `car-photos`.
3. Для простого учебного проекта можно сделать bucket публичным.
4. Загрузите изображения машин.
5. Скопируйте публичный URL файла.
6. Вставьте URL в поле `photo_url` у машины через админку или SQL.

## Частые проблемы

Если приложение показывает fallback-данные:

- Проверьте, что `.env` существует.
- Проверьте `VITE_SUPABASE_URL`.
- Проверьте `VITE_SUPABASE_ANON_KEY`.
- Перезапустите dev-сервер после изменения `.env`.

Если кабинет не показывает заказы:

- Пользователь должен быть авторизован.
- В `orders.user_id` должен быть UUID этого пользователя.
- RLS-политики из `schema.sql` должны быть применены.

Если фото машин не показываются:

- Проверьте, что `photo_url` открывается в браузере напрямую.
- Убедитесь, что Storage bucket публичный или URL доступен без авторизации.
- Проверьте, что ссылка ведет именно на изображение, а не на HTML-страницу.
